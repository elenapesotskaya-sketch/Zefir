# Техническое описание системы редактирования каталога

## Архитектура

Система редактирования состоит из следующих компонентов:

### Компоненты (Components)

#### `ProductCard.tsx`
Компонент отдельной карточки товара с поддержкой двух режимов:

**Режим просмотра (isEditing=false):**
- Отображает товар как обычно: изображение, название, описание, цена и статус

**Режим редактирования (isEditing=true):**
- Предоставляет редактируемые поля для:
  - Названия товара
  - Описания товара
  - Цены (EUR)
  - Статуса (выпадающий список: "В наличии" / "Закончилось")
  - Изображения (кнопка загрузки)
- Кнопка "Сохранить" (активна при наличии изменений)
- Отслеживание состояния через локальные useState хуки

#### `CatalogDisplay.tsx`
Главный компонент для отображения всех разделов каталога:

- Управляет состоянием всех товаров (`dynamicItems`)
- Загружает каталог с сервера при монтировании
- Передает режим редактирования (`isEditing`) в `ProductCard`
- Обрабатывает сохранение товаров через `handleSaveItem`
- Перезагружает каталог после каждого сохранения

#### `PasswordModal.tsx`
Модальное окно для ввода пароля:

- Предоставляет интерфейс ввода пароля
- Отправляет пароль на сервер для проверки
- Вызывает callback `onSuccess` при успешной проверке

### API Routes

#### `app/api/verify-password/route.ts`
Проверяет пароль:
- POST запрос с паролем
- Возвращает `{ valid: true/false }`

#### `app/api/catalog/route.ts`
Управляет каталогом:

**GET:**
- Возвращает текущий каталог из Blob Storage
- Формат: `{ items: CatalogItem[], lastUpdated: string }`

**POST:**
- Принимает: `{ password, action, item/items }`
- Действия:
  - `update-item` - обновляет/добавляет один товар
  - `bulk-update` - заменяет весь каталог
- Проверяет пароль перед сохранением
- Сохраняет в Vercel Blob Storage

#### `app/api/upload/route.ts`
Загружает изображения:
- POST с FormData (файл)
- Сохраняет в Vercel Blob Storage
- Возвращает: `{ url: string }`

## Поток данных

### Инициализация
```
1. page.tsx монтируется
2. CatalogDisplay загружает каталог через GET /api/catalog
3. Все товары отображаются в режиме просмотра
```

### Включение редактирования
```
1. Пользователь нажимает "Редактировать каталог"
2. Открывается PasswordModal
3. Пользователь вводит пароль
4. POST запрос на /api/verify-password
5. При успехе устанавливается isEditing=true в page.tsx
6. CatalogDisplay получает isEditing=true в props
7. Все ProductCard переходят в режим редактирования
```

### Редактирование и сохранение товара
```
1. Пользователь изменяет поля в ProductCard
2. handleFieldChange обновляет локальное состояние editData
3. hasChanges устанавливается в true (активирует кнопку)
4. Пользователь нажимает "Сохранить"
5. handleSave вызывает onSave (передано из CatalogDisplay)
6. CatalogDisplay.handleSaveItem отправляет POST на /api/catalog
7. API проверяет пароль и сохраняет в Blob
8. CatalogDisplay перезагружает каталог (GET /api/catalog)
9. Все компоненты обновляются с новыми данными
```

### Загрузка изображения
```
1. Пользователь нажимает иконку загрузки в ProductCard
2. handleImageUpload отправляет файл на /api/upload
3. API сохраняет файл в Blob и возвращает URL
4. ProductCard обновляет editData.imageUrl
5. Новое изображение отображается в карточке
6. При сохранении товара URL сохраняется в базу
```

## Хранилище (Vercel Blob)

### Структура файлов
- `catalog.json` - основной файл каталога
- Изображения сохраняются с автоматическими именами

### Структура данных каталога
```typescript
{
  items: [
    {
      id: string;           // Уникальный ID товара
      sectionId: string;    // bouquets | boxes | baskets | sets
      name: string;         // Название товара
      description: string;  // Описание
      price: number;        // Цена в EUR
      inStock: boolean;     // Статус доступности
      imageUrl: string;     // URL изображения
    }
  ],
  lastUpdated: string;      // ISO строка времени
}
```

## Раделы каталога

1. **bouquets** - Цветы в букете
2. **boxes** - Цветы в коробке
3. **baskets** - Цветы в корзинке-сумочке
4. **sets** - Наборы цветов и комплименты

## Безопасность

- **Пароль защита**: Пароль `1931` проверяется на сервере
- **Приватное хранилище**: Все файлы в Blob Storage имеют access=private
- **Проверка на каждое действие**: Пароль проверяется для каждого POST запроса
- **Токен в env**: BLOB_READ_WRITE_TOKEN хранится в переменных окружения

## Инициализация данных

Скрипт `scripts/init-catalog.js`:
- Проверяет есть ли уже каталог в Blob
- Если нет - создает новый каталог со всеми товарами
- Используется один раз при первой загрузке
- Можно запустить: `node scripts/init-catalog.js`

## Производительность

- **Локальное состояние**: Изменения отслеживаются локально до нажатия "Сохранить"
- **Ленивая загрузка**: Каталог загружается один раз при монтировании
- **Оптимизированное сохранение**: Сохраняется только при явном клике "Сохранить"
- **Перезагрузка после сохранения**: Гарантирует синхронизацию с сервером

## Тестирование
