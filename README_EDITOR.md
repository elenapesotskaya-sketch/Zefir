# Система редактирования каталога букетов

## Что было реализовано

Система позволяет редактировать каталог вкусных букетов **прямо на странице** без открытия отдельных окон и диалогов.

### ✨ Возможности

- **Инлайн редактирование** - редактируйте товары прямо на странице
- **Четыре раздела каталога**:
  - Цветы в букете
  - Цветы в коробке  
  - Цветы в корзинке-сумочке
  - Наборы цветов и комплименты
- **Редактируемые поля для каждого товара**:
  - Название товара
  - Описание (многострочный текст)
  - Цена в EUR
  - Статус (выпадающий список: "В наличии" / "Закончилось")
  - Изображение (загрузка с компьютера)
- **Защита паролем** - пароль: `1931`
- **Облачное хранилище** - данные сохраняются в Vercel Blob Storage
- **Автоматическое отслеживание изменений** - кнопка "Сохранить" активна только при изменениях

## Быстрый старт

### 1. Инициализация каталога (только первый раз)

```bash
node scripts/init-catalog.js
```

Это создаст базовый каталог со всеми товарами в Vercel Blob Storage.

### 2. Использование редактора

1. **Откройте приложение** - перейдите на страницу сайта
2. **Прокрутите до конца** - найдите кнопку "Редактировать каталог" в футере
3. **Введите пароль**: `1931`
4. **Редактируйте товары** - для каждой карточки:
   - Измените нужные поля
   - Нажмите кнопку "Сохранить"
5. **Выйдите из режима** - нажмите "Выход из редактирования"

## Архитектура системы

### Компоненты

| Компонент | Назначение |
|-----------|-----------|
| `ProductCard.tsx` | Карточка товара с режимом редактирования |
| `CatalogDisplay.tsx` | Отображение всех разделов каталога |
| `PasswordModal.tsx` | Модальное окно для ввода пароля |

### API маршруты

| Маршрут | Метод | Назначение |
|---------|-------|-----------|
| `/api/catalog` | GET | Получить весь каталог |
| `/api/catalog` | POST | Сохранить/обновить товары |
| `/api/verify-password` | POST | Проверить пароль |
| `/api/upload` | POST | Загрузить изображение |

### Хранилище

- **Vercel Blob Storage** - для каталога (catalog.json) и изображений
- **Приватное хранилище** - все файлы защищены и недоступны публично
- **Защита паролем** - пароль проверяется для каждого POST запроса

## Структура данных

### Товар (CatalogItem)

```typescript
{
  id: string;              // Уникальный ID (например: "bouquet-1")
  sectionId: string;       // Раздел: bouquets, boxes, baskets, sets
  name: string;            // Название товара
  description: string;     // Описание
  price: number;           // Цена в EUR
  inStock: boolean;        // true = "В наличии", false = "Закончилось"
  imageUrl: string;        // URL изображения в Blob Storage
}
```

### Каталог (CatalogData)

```typescript
{
  items: CatalogItem[];    // Массив всех товаров
  lastUpdated: string;     // ISO строка времени последнего обновления
}
```

## Производительность и оптимизация

- **Локальное состояние** - изменения отслеживаются локально до сохранения
- **Ленивая загрузка** - каталог загружается один раз при монтировании
- **Выборочное сохранение** - сохраняется только при явном клике
- **Перезагрузка после сохранения** - гарантирует актуальность данных

## Переменные окружения

Необходимо добавить в `.env.local`:

```
BLOB_READ_WRITE_TOKEN=<ваш токен>
```

Токен генерируется автоматически при подключении Vercel Blob Storage к проекту.

## Примеры использования

### Редактирование названия товара

1. Перейдите в режим редактирования (пароль: 1931)
2. В текстовом поле "Название" замените текст
3. Нажмите "Сохранить"

### Изменение цены

1. Откройте режим редактирования
2. В числовом поле "Цена (EUR)" введите новое значение
3. Нажмите "Сохранить"

### Изменение статуса товара

1. Откройте режим редактирования
2. В выпадающем списке "Статус" выберите:
   - "В наличии" - товар доступен (зеленый статус)
   - "Закончилось" - товар нет в наличии (серый статус)
3. Нажмите "Сохранить"

### Загрузка нового изображения

1. Откройте режим редактирования
2. Нажмите на иконку загрузки (левый нижний угол изображения)
3. Выберите новое изображение с компьютера
4. Изображение загружается автоматически
5. Нажмите "Сохранить" чтобы сохранить товар с новым изображением

## Безопасность

✅ **Пароль защита** - все операции редактирования требуют правильный пароль
✅ **HTTPS только** - Blob Storage использует защищенные соединения
✅ **Приватное хранилище** - файлы недоступны публично
✅ **Проверка на сервере** - пароль всегда проверяется на сервере
✅ **BLOB_READ_WRITE_TOKEN** - специальный токен для доступа к хранилищу

## Отладка

### Каталог не загружается

1. Проверьте переменную окружения `BLOB_READ_WRITE_TOKEN`
2. Запустите `node scripts/init-catalog.js`
3. Проверьте консоль браузера на ошибки (F12)

### Пароль не работает

1. Убедитесь что вводите: `1931`
2. Проверьте подключение к интернету
3. Откройте консоль браузера и проверьте запросы к `/api/verify-password`

### Изображения не загружаются

1. Проверьте размер файла (должен быть разумный)
2. Убедитесь что формат поддерживается (JPEG, PNG, WebP)
3. Проверьте токен Blob Storage в переменных окружения

## Контакты и поддержка

При возникновении проблем:
1. Проверьте файлы документации (INLINE_EDITOR_GUIDE.md, TECHNICAL_SETUP.md)
2. Откройте консоль разработчика (F12) и проверьте ошибки
3. Убедитесь что переменные окружения установлены правильно
